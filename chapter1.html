Chapter 1: Computational Thinking


How Computers Help Us Think

The 4 main constructs of Computational Thinking are: Decomposition, Abstraction, Pattern Recognition and Algorithms:

Decompostion - breaking down any problem into a lot more managable chunks to solve.

  • Advantages
    • It means that you're not overwhelmed by the amount of work it seems is require, if you do it piece by piece.
    • Means you do exactly what is required, as decomposition requires some planning, which will make the end result more accurate to the target, compared to ‘going with the flow’.

  • Disadvantages
    • Solving a problem in chunks may cause you to drift away from the overall goal, and therefore the chunks don’t fit together at the end quite the way they should.
    • Decomposition assumes that the whole solution to the problem is known in advance. And often this a problem is given to, design and work out the solution.
Object Oriented Programming is another way of breaking down problems and even the functionality at the same time. An object / class often contains lots of data. These objects are isolated from each other, which avoids interference problems, and it also gives the opportunity to reuse the same object data in the future for other problems.

Abstraction - ignoring any irrelevant data and picking out / using only the relevant data. This is often used so that a programmer is not distracted from the main objective of a project, and stays on track.

Pattern Recognition - looking for similarities among different problems and solving them in similar ways to before.

Algorithms - a step by step solution to a problem, or the rules in which to follow to solve the problem.
Recursion - is where something is defined in times of itself. A good example is using a function name within the same functions definition, to restart the function, this can cause infinite loops within the function.
An example of this is the GNU operating system. GNU is a recursive acronym is an acronym that refers to itself. This is applicable as GNU stands for GNU's Not Unix.
